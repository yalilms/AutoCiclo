1. INTRODUCCIÓN

1.1. Descripción del proyecto

AutoCiclo es un sistema de gestión integral desarrollado específicamente para desguaces de automóviles. El proyecto surge como respuesta a la necesidad de modernizar y optimizar los procesos de control de inventario en este tipo de establecimientos, donde la gestión eficiente de vehículos y piezas es fundamental para la rentabilidad del negocio.

La aplicación permite llevar un control exhaustivo de los vehículos que ingresan al desguace, las piezas que se extraen de ellos, su ubicación en el almacén, el stock disponible y la generación de informes detallados.

1.2. Objetivos

Los objetivos principales del proyecto AutoCiclo son los siguientes:

  • Desarrollo Profesional: Crear una aplicación de escritorio robusta utilizando JavaFX como framework de interfaz gráfica.
  • Gestión de Datos: Implementar un sistema completo de gestión CRUD.
  • Experiencia de Usuario (UX): Diseñar una interfaz moderna e intuitiva.
  • Persistencia: Integrar una base de datos MySQL.
  • Funcionalidad Avanzada: Proporcionar búsqueda en tiempo real y estadísticas.
  • Informes: Generar documentación profesional en PDF y HTML.
  • Arquitectura: Aplicar estrictamente el patrón MVC.

1.3. Alcance

El sistema AutoCiclo abarca las áreas de gestión de vehículos, piezas, trazabilidad, análisis y reportes, proporcionando una solución integral al usuario.

2. TECNOLOGÍAS UTILIZADAS

2.1. Lenguajes y Frameworks

Java 24 y JavaFX 25 formaron la base del desarrollo, seleccionados por su robustez y capacidades modernas de interfaz gráfica.

2.2. Herramientas de Desarrollo

Se utilizaron Gradle 8.8 para la gestión del ciclo de vida y dependencias, MySQL 8.0 para la persistencia, y Scene Builder para el diseño visual.
